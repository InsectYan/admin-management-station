# 小说创作平台代码开发设计报告

## 项目概述

本报告旨在设计一个基于 React+Ant Design+Ant Design Charts 的 Web 端小说创作平台，该平台将作为微前端(qiankun)架构下的子应用，通过Node.js鸡蛋框架与PostgreSQL数据库构建完整的技术栈。

**多应用并行**：本子应用 `app_key=novel`，须遵循 **[应用端口与命名注册表.md](./应用端口与命名注册表.md)** — API **7002**、Vite dev **5174**、Docker 前端 **8081**、数据库 **`novel_db`**。

## 技术架构

### 技术栈选择与优势

| 技术组件 | 选择理由 | 主要功能 |
|---------|---------|---------|
| **Docker** | 环境一致、易于部署扩展 | 容器化前后端与数据库，Nginx 托管静态资源 |
| **React** | 现代化前端框架，组件化开发，丰富的生态系统 | 构建用户界面，管理组件状态，实现交互逻辑 |
| **Ant Design** | 企业级UI组件库，完整的布局组件，与React无缝集成 | 提供标准化UI组件，保证界面一致性，实现美观的UI效果 |
| **qiankun** | 微前端标准解决方案，轻量级，与React/Vue等框架兼容 | 实现子应用与主应用的集成，处理生命周期管理，解决路由冲突 |
| **Node.js** | 轻量级、高性能、异步I/O模型 | 构建RESTful API，处理业务逻辑，连接PostgreSQL数据库 |
| **PostgreSQL** | 强事务支持，JSONB类型，高级查询功能 | 存储小说内容，支持复杂数据结构，保证数据一致性 |

### 系统架构设计

![系统架构设计图](https://example.com/architecture.png)

系统采用分层架构设计，分为以下四个层次：

1. **表现层**：基于React+Ant Design构建用户界面，实现小说管理、详情/编辑和新建小说功能
2. **业务逻辑层**：通过Node.js鸡蛋框架处理业务逻辑，包括小说创建、编辑、删除和查询等操作
3. **数据访问层**：通过TypeORM或Knex.js等ORM工具实现与PostgreSQL数据库的交互
4. **基础设施层**：Docker 容器编排、Nginx 静态托管、HTTP API 通信、qiankun 微前端集成

### 交互流程设计

#### 小说管理交互流程

![小说管理交互流程](https://example.com/novel-management.png)

1. 用户通过左侧菜单点击"小说管理"进入小说列表页面
2. 用户可在顶部筛选栏输入小说名称或选择小说状态进行筛选
3. 系统向后端发送筛选请求，获取分页数据
4. 用户点击"新建"按钮，跳转到新建小说步骤条页面
5. 用户在步骤条页面填写基本信息，点击下一步
6. 用户填写大纲信息，点击下一步
7. 用户填写正文创作信息，点击下一步
8. 用户进行审稿润色，点击完成
9. 系统将小说数据存储到PostgreSQL数据库
10. 用户点击小说列表中的"查看"按钮，跳转到小说详情/编辑页面

#### 新建小说步骤条交互流程

![新建小说步骤条交互流程](https://example.com/creation-process.png)

1. 用户点击"新建"按钮，进入步骤条引导界面
2. 用户填写基本信息（名称、立意等），点击下一步
3. 系统暂存表单数据到 sessionStorage 或后端草稿 API
4. 用户进入大纲构建界面，系统从暂存数据中加载已填写的信息
5. 用户填写正文创作内容，点击下一步
6. 系统暂存表单数据到 sessionStorage 或后端草稿 API
7. 用户进入审稿润色界面，系统从暂存数据中加载已填写的信息
8. 用户点击"完成"按钮，系统将所有步骤数据提交到后端
9. 后端验证数据完整性，存储到PostgreSQL数据库
10. 系统返回小说管理列表，显示新创建的小说

## 数据库设计

### 表结构设计

#### 小说表(novels)

```sql
-- 创建小说表
CREATE TABLE novels (
  id SERIAL PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  author_id INT NOT NULL REFERENCES users(id),
  status VARCHAR(50) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  plot JSONB, -- 存储小说大纲等复杂结构
  progress INT DEFAULT 0, -- 创作进度百分比
  word_count INT DEFAULT 0, -- 总字数统计
  draft JSONB -- 存储草稿信息
);

-- 创建索引
CREATE INDEX idx_novel_title ON novels (title);
CREATE INDEX idx_novel_status ON novels (status);
CREATE INDEX idx_novel_author ON novels (author_id);
CREATE INDEX idx_novel_plot ON novels USING GIN (plot); -- 为JSONB字段创建GIN索引
```

#### 章节表(chapters)

```sql
-- 创建章节表
CREATE TABLE chapters (
  id SERIAL PRIMARY KEY,
  novel_id INT NOT NULL REFERENCES novels(id) ON DELETE CASCADE,
  title VARCHAR(255) NOT NULL,
  content TEXT NOT NULL,
  chapter_number INT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建索引
CREATE INDEX idx_chapter_novel ON chapters (novel_id);
CREATE INDEX idx_chapter_number ON chapters (novel_id, chapter_number);
```


### 数据库优化策略

1. **索引优化**：
   - 为频繁查询的字段（如`title`、`status`）创建B-tree索引
   - 为JSONB字段（如`plot`）创建GIN索引，支持复杂查询
   - 为外键字段（如`author_id`）创建索引，加速关联查询

2. **查询优化**：
   - 使用`EXPLAIN ANALYZE`分析查询性能
   - 避免在JSONB字段上使用`->`操作符进行模糊查询
   - 对于频繁更新的字段，使用`@>`操作符进行存在性查询

3. **事务管理**：
   - 对于创建/更新小说等操作，使用事务保证数据一致性
   - 设置合理的事务隔离级别（默认read committed）
   - 对于读多写少的场景，考虑使用物化视图优化查询性能

```typescript
export const createLangChainAgent = async () => {

  // 创建工具列表

        return response;
      },
      description: "生成小说的立意和主题"

    })

  // 创建零次射击代理
  const agent = ZeroShotAgent.fromLLMAndTools(llm, tools, {
```

```typescript
export class PersistenceService {
  async saveNovel(novel: Novel): Promise<void> {
    // 保存到数据库
    await this.novelRepository.save(novel);

    // 保存到sessionStorage
    novelStore.set(`novel:${novel.id}`, novel);
  }

  async loadNovel(id: string): Promise<Novel | null> {
    // 从sessionStorage加载
    const storedNovel = novelStore.get(`novel:${id}`);
    if (storedNovel) {
      return storedNovel;
    }

    // 从数据库加载
    const dbNovel = await this.novelRepository One({ where: { id } });
    if (dbNovel) {
      // 存储到sessionStorage
      novelStore.set(`novel:${id}`, dbNovel);
      return dbNovel;
    }

    return null;
  }
}
```

```typescript
// 防抖函数
export const debounce = (func: Function, wait: number): Function => {
  let timeout: NodeJS.Timeout | null = null;

  return (...args: any[]) => {
    if (timeout) {
      clearTimeout(timeout);
    }

    timeout = setTimeout(() => {
      func(...args);
    }, wait);
  };
};

// 使用防抖保存小说进度
export const saveNovelProgress = debounce(async (novelId: string, progress: number) => {
  try {

    // 更新sessionStorage中的进度

      novelStore.set(`novel:${novelId}`, { ...novel, progress });
    }
  } catch (error) {
```

## 总结
