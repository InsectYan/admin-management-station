# 应用端口与命名注册表



> **规范来源**：`.cursor/rules/app-registry.mdc` · **架构**：`.cursor/rules/app-self-contained.mdc` · **缓存**：`.cursor/rules/cache-local.mdc`  

> 每个应用**自带 Postgres**；缓存默认 **memory**（无需 Redis 容器）。



## 端口段约定

| 段 | 用途 | 示例 |
|----|------|------|
| **4xxx** | Agent 平台 API（`agent-management-master`） | **4001** |
| **51xx** | AMS 子应用 Vite dev | 5100 main, 5102 testgen |
| **52xx** | AMS 子应用 BFF API | 5200 main, 5202 testgen |
| **53xx** | AMS 子应用 Postgres 宿主机映射 | 5300 main, 5302 testgen |
| **5500** | Agent 平台 Postgres（独立，非 53xx 递增） | agent-postgres |
| **3001** | **fitness-agent / cartoon-agent BFF**（非 Agent 平台） | 与 4001 并行无冲突 |

连接 Agent 平台：`AGENT_PLATFORM_URL=http://127.0.0.1:4001`（Docker 内 `host.docker.internal:4001`）。

菜单 API `/api/menus`、`/api/menus/root` 会附带 `entry`、`subapp` 字段；可用 `SUBAPP_{APP_KEY}_ENTRY` 覆盖。

子应用注册由 `ams-main local` 启动时自动执行 `sync-subapps.mjs` 扫描 `project-sub/*/subapp.manifest.json` 完成，无需手动改 `init.sql`。

## 新增子应用



1. 复制 `project-sub/novel-sub/` 整目录到 `project-sub/{name}-sub/`

2. 在本表登记 API/PG 宿主机端口（递增）

3. 编写 `subapp.manifest.json` 并执行 `ams-main sync:subapps`（或启动主应用自动同步）

4. 按 [`.cursor/rules/subapp-onboarding.mdc`](../.cursor/rules/subapp-onboarding.mdc) 完成接入 checklist

5. 实现完整 `{app}/deploy/docker-compose.yml` 并验证可独立 `ams-{app_key} local`
