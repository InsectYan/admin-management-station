# 应用端口与命名注册表

> **规范来源**：`.cursor/rules/app-registry.mdc`  
> **场景**：主应用、多个子应用、Agent **并行**本地开发或 Docker 全栈联调时，避免端口冲突与数据库混用。

## 1. 为什么需要注册表

本平台不是单体应用：菜单基座、各业务子应用、Agent 服务会**同时**启动。若共用默认端口（如全部 7001、全部 `postgres` 默认库），会导致：

- Vite dev server 端口冲突
- Egg.js BFF 监听冲突
- 子应用连错数据库、迁移污染其他域

**凡涉及端口、数据库名、容器名的设计与实现，必须先查本表。**

## 2. 已登记应用

| app_key | 说明 | 代码目录 | Compose 目录 | API 端口 | 前端 Docker | Vite dev | 数据库名 |
|---------|------|----------|--------------|----------|-------------|----------|----------|
| `main` | 主应用（菜单 + Qiankun） | `menu-master/` · `backend`/`frontend` | `menu-master/docker-compose.yml` | 7001 | 8080 | 5173 | `admin_platform` |
| `novel` | 小说子应用 | `apps/novel-*` | `apps/novel/docker-compose.yml` | 7002 | 8081 | 5174 | `novel_db` |
| `agent` | Pi Agent | `apps/agent-server/` | `apps/agent-server/docker-compose.yml` | 7003 | — | 7003 | `admin_platform` |

### 共享基础设施

| 组件 | 容器名 | 端口 |
|------|--------|------|
| PostgreSQL | `ams-postgres` | 5432 |
| Redis | `ams-redis` | 6379 |

## 3. 命名约定

| 类型 | 格式 | 示例 |
|------|------|------|
| 应用标识 | 小写 `app_key` | `main`, `novel` |
| API 容器 | `ams-api-{app_key}` | `ams-api-main` |
| 前端容器 | `ams-{app_key}-frontend` | `ams-novel-frontend` |
| 数据库 | 语义名或 `{app_key}_db` | `admin_platform`, `novel_db` |
| 环境变量前缀 | `{APP_KEY}_` 大写 | `MAIN_PORT`, `NOVEL_POSTGRES_DB` |
| Vite 变量 | `VITE_*` + `VITE_APP_KEY` | `VITE_API_BASE`, `VITE_DEV_PORT` |

## 4. 环境变量模板

### deploy/config/.env.local（团队共享）

```env
POSTGRES_USER=admin
POSTGRES_PASSWORD=admin123

MAIN_PORT=7001
MAIN_POSTGRES_DB=admin_platform

NOVEL_PORT=7002
NOVEL_POSTGRES_DB=novel_db

AGENT_PORT=7003

JWT_SECRET=CHANGE_ME_LOCAL_JWT
```

### 主应用 frontend（menu-master/frontend/.env.example）

```env
VITE_APP_KEY=main
VITE_API_BASE=http://localhost:7001/api
VITE_DEV_PORT=5173
VITE_SUBAPP_NOVEL_ENTRY=http://localhost:5174
```

### 小说子应用 frontend（apps/novel-frontend/.env.example）

```env
VITE_APP_KEY=novel
VITE_API_BASE=http://localhost:7002/api
VITE_DEV_PORT=5174
```

### 主应用 backend（menu-master/backend/.env.example）

```env
APP_KEY=main
PORT=7001
POSTGRES_HOST=127.0.0.1
POSTGRES_PORT=5432
POSTGRES_USER=admin
POSTGRES_PASSWORD=admin123
POSTGRES_DB=admin_platform
```

## 5. 宿主机并行开发示例

```bash
# 终端 1：基础设施
ams local:infra

# 终端 2：主应用 API :7001
cd menu-master/backend && npm run dev

# 终端 3：主应用前端 Vite :5173
cd menu-master/frontend && npm run dev

# 终端 4：小说 API :7002
cd apps/novel-backend && npm run dev

# 终端 5：小说前端 Vite :5174
cd apps/novel-frontend && npm run dev

# 终端 6：Agent :7003
cd apps/agent-server && npm run dev
```

浏览器访问主应用：`http://localhost:5173`（开发）或 `http://localhost:8080`（Docker）。

## 6. 新增子应用

按顺序：

1. 在本表与 `.cursor/rules/app-registry.mdc` 增加一行（端口取下一个：API 7004、Vite 5175、Docker 8082…）
2. 创建 `{app_key}_db` 并在 `deploy/config/.env.local` 增加 `{APP_KEY}_POSTGRES_DB`
3. 按 [`.cursor/rules/docker-compose.mdc`](../.cursor/rules/docker-compose.mdc) 创建 compose 文件，并更新 `deploy/docker-compose.yml` include
4. 主应用 `menu_items` 注册 `microapp_name`；Qiankun entry 指向该子应用 Vite/Docker 地址
